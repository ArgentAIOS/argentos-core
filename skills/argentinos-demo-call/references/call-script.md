# Live Demo Call Script (60 Minutes)

## Pre-Call (5 Minutes Before Start)

Collect:

- Company name
- Industry
- Team size/headcount
- Primary contact and role
- Known pain points
- Existing systems (ticketing, email, chat, monitoring)

Inject context in one concise block before call start.

## Act 1: Introduction (0:00-0:05)

Goal:

- Build trust quickly.
- Set expectation that this is interactive system design.

Script:

- "Thanks for joining. I reviewed what Jason shared, but I want your version first."
- "Walk me through a normal Tuesday from opening to close. Where does it break?"

## Act 2: Discovery (0:05-0:25)

Goal:

- Extract operational pain with concrete examples.

Required captures:

- Trigger event
- Current manual handler
- Time cost
- Failure mode
- Escalation path

Use probe sets from `probe-questions.md`.

## Act 3: Proposal (0:25-0:40)

Goal:

- Map pain points to a proposed agent roster live.

Structure:

1. Restate top pain points.
2. Present candidate agents with one-line purpose.
3. Ask confirmation:
   - "Is this the right split?"
   - "What did I miss?"

Use phrasing from `proposal-language.md`.

## Act 4: Live Build (0:40-0:55)

Goal:

- Build artifacts live while narrating value.

Generate in order:

1. Strategy document
2. Technical implementation spec
3. Bootstrap prompt

Narrate with `live-build-narration.md`.

## Act 5: Close (0:55-1:00)

Goal:

- Lock phase-1 anchor and ownership.

Ask:

- "What one automation on day one would change your week?"

Confirm:

- First deployment target
- Timeline window
- Approvals needed

Use closing options in `close-questions.md`.
