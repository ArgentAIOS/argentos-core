# Live Demo Run Checklist And Output Template

## In-Call Checklist

Before call:

- Pre-call context block captured
- Dashboard + orb visible
- DocPanel ready for live notes

During call:

- Top three pain points captured with concrete examples
- Proposed roster confirmed by customer
- Three artifacts drafted live
- Day-one anchor question asked

After call:

- Handoff summary produced
- Next-step owner and timeline captured
- Follow-up risks and blockers captured

## Required Handoff Output

Return this exact structure at the end of each call:

## Failure Conditions

Do not mark complete if:

- Pain points are generic and not tied to concrete workflow failures.
- Artifacts are promised but not drafted.
- No phase-1 anchor is captured.
