# Build Phasing Template

## Phase 1 (2-4 Weeks)

Objective:

- Deploy one high-impact workflow end to end.

Typical scope:

- Core roster subset
- Critical integrations only
- Baseline guardrails
- Initial reporting

Exit criteria:

- Day-one anchor automation live
- Stakeholder acceptance
- Known incidents tracked

## Phase 2 (2-3 Weeks)

Objective:

- Expand operational coverage and add quality controls.

Typical scope:

- Additional agents
- More channels/integrations
- Deeper policy controls
- Expanded analytics

Exit criteria:

- Reduced manual handoff volume
- Reliable escalations
- Stable reporting cadence

## Phase 3 (4-8 Weeks)

Objective:

- Platform maturity and scaled automation.

Typical scope:

- Advanced workflows
- Form/intake automation
- Marketplace-ready packaging
- Governance hardening

Exit criteria:

- Repeatable onboarding pattern
- Documented runbooks
- Measurable ROI outcomes
