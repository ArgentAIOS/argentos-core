# Five Discovery Phases

## Phase 1: Business Context

Objective:

- Understand operating reality and business constraints.

Required outputs:

- Normal daily workflow summary
- Core stakeholder map
- Decision cadence

## Phase 2: Pain Points

Objective:

- Convert complaints into specific operational failure patterns.

Required outputs:

- Trigger, actor, failure, impact for each pain point
- Frequency and severity estimates

## Phase 3: Agent Identification

Objective:

- Propose a minimal roster that maps directly to validated pain.

Required outputs:

- Candidate agent list
- Role-to-pain mapping
- Phase-1 vs later-phase split

## Phase 4: Systems Audit

Objective:

- Verify practical implementation path.

Required outputs:

- Integration inventory
- Data source access plan
- Non-negotiable dependencies

## Phase 5: Alignment And Guardrails

Objective:

- Define boundaries, approvals, and failure handling.

Required outputs:

- Never-do constraints
- Human approval checkpoints
- Escalation matrix
